HTML Structure

The basic page structure contains:

<body>
   
   <!-- This class is mainly used to control responsiveness across different devices -->  
   <div class=”mobile-wrapper”>

       <!-- This is the main stack in which pages get loaded through ajax requests -->
       <div id=”main-stack”>

           <!-- Here is contained the navigation bar -->
           <div class=”navbar”></div>

           <!-- This is the body -->
           <div class=”body”></div>

       </div>
   </div>
</body>

The App Life-cycle Workflow

The app loads pages as requested using asynchronous  requests. It prevents the default href links to load directly  in to the browser and hooks the event in which an ajax request is processed and finally the app then displays the page previously requested in a smooth way using loading screens. This way the whole  user experience is smoother and simpler.

The app Life-cycle is as follows:

  1. User clicks on a link
  2. App hooks such event
  3. The requested URL is taken and then utilised to load the content through ajax
  4. Loading screens are shown while the page is being loaded
  5. The default click event is prevented so the page doesn't reload
  6. The current page’s content is replaced with the new content

CSS  Styling

Washington uses normalize.css in order to reset the default browser styles. The app styling is located in a separate css file.

How to Use It

  • How to Set Up Washington:

  • Icons

In order to add a new icon the following syntax needs to be used:

<div class=”icon [icon name]”></div>

Washington uses icons from ionicons.com. The list of icons you are able to use as well as the class that needs to be used in [icon name] can be found in the following web www.ionicons.com. This is a real example of an icon:

<div class=”icon ion-ios-home-outline”></div>

Please note, in order to change an icon in the web, it is necessary to replace the icon class in [icon name].  There are more than 730 icons available and ready for you to use.

  • Ajax Links

The app handles automatically the ajax requests so no javascript code needs to be written. All that needs to be done in order to enable the page to load dynamically through ajax, is to add the following attribute to the <a> tag.

data-load=1

An example can be found below:

<a href=”login.html” class=”button” data-load=”1”></a>

  • Maps

Washington gives you the possibility to add a map. If you want to do so, you would have to create a widget map using a div and the  “w-widget w-widget-map” classes, you can find the available options below which needs to be added as attributes in the div:

  • Cordinates

data-widget-latlng=”[latitude, longitude]”

This is the map cordinates.

  • Map Style

data-widget-style=”[style]”

Style options available are:

  • roadmap
  • satelite
  • hybrid
  • terrain

  • Map Zoom

data-widget-zoom=”[zoom]”

  • Zoom levels  can be set from 0 to 18.

  • Map Tooltip

data-widget-tooltip=”[tooltip]”

  • Any text you want to display in the map tooltip can be specified here.

  • Disable Touch

data-widget-touch=”[0 or 1]”

  • 0 disables touch and 1 enables it. This option allows you to scroll down the page without touching the map and moving it around.

An example of a map can be found below:

<div class="w-widget w-widget-map square-map" data-widget-latlng="51.752021,-1.257726" data-widget-style="satellite" data-widget-zoom="18" data-widget-tooltip="Headquarters" data-disable-touch="1"></div>

  • Google Fonts

This is the code that can be used to load Google Fonts.

<script>

    WebFont.load({

      google: {

        families: ["Montserrat:400,700"]

      }

    });

  </script>

If you want to change the font that has been used in the app, you would have to replace the name of the font in “families” (i.e: where it says [“Montserrat: 400,700”]).

  • Infinite Animation

data-infinite=”[ 0 or 1 ]”

  • If 1 is selected then slides will be scroll infinitely.  

  • Animation Delay

data-delay=”[delay]”

  • The value specified in duration needs to be specified in milliseconds. That determines the delay of the slide before moving forward to the next one .

  • Autoplay

data-autoplay=”[ autoplay ]”

  • If 1 is selected then slides will play automatically.

  • Stop Autoplay After

data-autoplay-limit=”[ limit ]”

  • It enables you to specify the number of slides that should have passed before stopping the autoplay. If 0 is specified, autoplay will never stop.
